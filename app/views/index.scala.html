This Web Application has been designed as part of my thesis, in order to allow me test how Lucene, Solr and ElasticSearch perform when searching using non-English languages.

The languages being tested are: French, German, Spanish and Chinese. A different index needs to be used for each language.

When creating the indices, a pre-defined set of test data was used for each language, that allows testing of features during a search such as stemming, accent removal, stopword removal, wildcard queries, and ranking of results. Please see the Appendix in the thesis report to see what test data has been used.

Lucene

Lucene is a library of classes for implementing a search server. This application contains the functionality for creating the indexes for the 4 languages under test, by giving it the directory location where the test text documents for each language are stored.

The indexes have already been set up, and so only need to be recreated if additional test data was required.

View Lucene Search Page »

Solr

Solr is a standalone search server, built on Lucene. It has been deployed on Tomcat7 on a separate Amazon EC2 instance. The default out-of-the-box analyzers have been used for indexing and querying of data.

Some customized analyzers have also been created, and can be used for searching on the Solr search page.

ElasticSearch

ElasticSearch is a standalone search server, built on Lucene. It has been deployed on a Netty server on a separate Amazon EC2 instance. The default out-of-the-box analyzers have been used for indexing and querying of data.
